# Research Notes

This directory is for dated research notes, observed run results, strategy conclusions, and environment findings that are useful during iteration but are not themselves frozen submission artifacts.

Use these locations consistently:

- `experiments/`: experiment design docs, hypotheses, pass/fail gates, and planned variants
- `docs/research/`: observed results, reproduction notes, environment issues, and research conclusions
- `records/`: frozen runnable snapshots for non-record and record-track runs

Recommended note format:

- use one file per meaningful run or finding
- prefix the filename with the date
- use `draft_notes` in the filename when the note is provisional
- include the exact command, hardware, environment, key metrics, and interpretation
- link to the raw local log when available

These notes can describe local development runs even when current `code bytes` accounting is not yet submission-accurate.
